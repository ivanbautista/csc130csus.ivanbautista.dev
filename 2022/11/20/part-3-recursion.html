Part 3: Recursion

Recursion is a programming technique in which a function calls itself to solve a problem. Recursion can be a powerful tool for solving problems, but it can also be difficult to understand and debug.

Some key concepts related to recursion include:

  • Variable scope: In programming, a variable’s scope refers to the parts of the program where the variable can be accessed. In a recursive function, each time the function calls itself, a new set of variables is created with their own scope.

  • System stack: When a function is called in a program, the details of the function call (such as the function’s arguments and local variables) are saved on the system stack. When the function returns, the details are removed from the stack. In a recursive function, each time the function calls itself, a new set of details is added to the stack.

  • System heap: The system heap is a portion of memory used to store dynamically allocated data. In a recursive function, the system heap may be used to store data that is needed across multiple recursive calls.

  • Loitering: Loitering occurs when data is no longer needed but is not properly deallocated, causing it to occupy space in memory unnecessarily. In a recursive function, loitering can occur if data is allocated on the heap but is not properly deallocated before the function returns.

  • Pools: A pool is a data structure that allows data to be reused rather than being deallocated and reallocated each time it is needed. In a recursive function, pools can be used to reduce the overhead of allocating and deallocating data on the heap.

Code of a recursion in JS

You can copy and paste the code into a JavaScript console from the web browser using the following key combination “Ctrl + Shift + i”, then paste the following code.

function countDown(n) {
  // Print the current value of n to the console
  console.log(n);

  // If n is greater than zero, call the function again with n - 1
  if (n > 0) {
    countDown(n - 1);
  }
}

Testing the function

//calls the functoin
countDown(5);
